Explicación del código

1. Constructores (__init__)
- Se llaman automáticamente cuando se crea una instancia de una clase
- Se utilizan para inicializar los atributos del objeto
- En nuestro programa:
  - ArchivoLogger.__init__: Abre un archivo y escribe un mensaje inicial
  - ConexionSimulada.__init__: Establece una conexión simulada
  - SistemaSensor.__init__: Inicializa un sistema de sensores

2. Destructores (__del__)
- Se llaman automáticamente cuando un objeto es destruido
- Útiles para liberar recursos (archivos, conexiones, etc.)
- En nuestro programa:
  - ArchivoLogger.__del__: Cierra el archivo y escribe un mensaje final
  - ConexionSimulada.__del__: Cierra la conexión simulada
  - SistemaSensor.__del__: Apaga todos los sensores

3. Cuándo se activan los destructores
- Cuando el objeto sale de ámbito (ej: fin de una función)
- Cuando se usa del objeto explícitamente
- Cuando el recolector de basura de Python elimina el objeto
- Al finalizar el programa

4. Buenas prácticas implementadas
- Los destructores verifican que los recursos existan antes de liberarlos
- Uso de hasattr() para evitar errores si el objeto no se inicializó completamente
- Manejo adecuado de excepciones potenciales
- Código bien comentado y estructurado

5. Ejecución del programa
Para ejecutar el programa:

python constructores_destructores.py


El programa generará archivos de log que demuestran el ciclo de vida completo de los objetos, mostrando claramente cuándo se llaman los constructores y destructores.
